# War Trade Simulator

A quantitative signal system that uses the relationship between equity prices and oil price momentum to time stock purchases and sales during geopolitical conflict periods. Includes a live daily signal dashboard, a historical backtest across seven major war events, and a Monte Carlo forecast engine.

---

## Project Overview

The core idea: during wars and geopolitical crises, oil price spikes are leading indicators of equity drawdowns. By watching how far a stock has fallen from its recent high (drawdown) and how aggressively oil is moving (spike relative to its 5-day average), you can size positions more intelligently than simply buying and holding.

The system runs three strategies side-by-side so you can compare their behaviour over the same market conditions:

| Strategy | Description |
|---|---|
| **OilWar Active** | Rebalances daily to a target allocation driven by drawdown + oil signals. Both buys and sells. |
| **OilWar Buy-Only** | Uses the same signals to decide *when* and *how much* to buy, but never sells. Tracks unrealized mark-to-market value. |
| **Buy & Hold** | 100% invested from day one. Passive benchmark. |

## Running the Historical Backtest

```bash
python3 backtest.py
```

Fetches data for seven conflict windows (Gulf War 1990 through Gaza 2023), runs all three strategies across each window, and writes `backtest_report.html` — a fully self-contained file you can open in any browser.

The report shows:
- **Overview tab** — side-by-side return, max drawdown, and Sharpe for every strategy across every event
- **Per-event tabs** — three-line equity chart, OilWar Active allocation bars, Buy-Only cumulative deployment chart, and an unrealized-value callout for the Buy-Only strategy

**Adding a new conflict window** — edit the `EVENTS` dict in `backtest.py`:

```python
EVENTS = {

    #                 ^fetch_start   ^bt_start     ^bt_end        ^color
    # fetch_start should be ~6 weeks before bt_start to warm up the 20-day lookback
}
```

## The Signal Algorithm

All OilWar strategies share the same underlying logic, defined once in `strategy.py`:

### Step 1 — Base allocation from drawdown

How far has the equity fallen from its 20-day high?

| Drawdown | Base allocation |
|---|---|
| < 3% | 0% (hold cash) |
| 3 – 5% | 25% |
| 5 – 7% | 50% |
| 7 – 10% | 75% |
| > 10% | 100% |

### Step 2 — Oil spike modifier

How aggressively is oil moving relative to its 5-day average?

| Oil spike vs 5-day avg | Effect |
|---|---|
| < 5% | × 0.50 (oil is quiet — reduce) |
| 5 – 10% | × 1.00 (neutral) |
| 10 – 20% | × 1.20 (moderate conflict signal — add) |
| > 20% | × 0.70 (extreme spike — pull back) |

### Step 3 — Momentum boost

If the equity has gained more than 1.5% over the past 3 days AND oil is calm (spike < 10%), add 25% to the allocation (capped at 100%).

## `strategy.py` — Architecture Reference

### `Signals` dataclass

Every strategy receives a frozen `Signals` object on each bar:

```python
@dataclass(frozen=True)
class Signals:
    price:      float   # equity close
    high_20d:   float   # 20-day rolling high
    drawdown:   float   # (high_20d - price) / high_20d
    return_3d:  float   # (price / price_3d_ago) - 1
    oil_price:  float   # oil close
    oil_5d_avg: float   # 5-day rolling oil mean
    oil_spike:  float   # (oil_price - oil_5d_avg) / oil_5d_avg
```

Strategies are pure functions of this object. They cannot see the calendar, portfolio balance, or anything outside these seven numbers.

### `Strategy` base class

```python
class Strategy:
    name:  str = "Unnamed Strategy"   # shown in charts and reports
    color: str = "#888888"            # hex colour for chart lines

    def next_allocation(self, signals: Signals) -> float:
        """Return desired allocation in [0.0, 1.0] for this bar."""
        raise NotImplementedError

    def reset(self) -> None:
        """Called before each new backtest window or history replay."""
        pass

    def on_bar(self, signals: Signals) -> None:
        """Hook called after accounting. Override for per-bar side effects."""
        pass
```

### `REGISTRY`

The list of strategy instances that `dashboard.py` and `backtest.py` both iterate:

```python
REGISTRY: list = [
    OilWarStrategy(),
    BuyOnlyOilWarStrategy(),
    BuyAndHoldStrategy(),
]
```

Every script that imports `REGISTRY` automatically picks up any strategies added here.

### Engine functions

```python
run_history(
    eq_series,           # pd.Series of equity closes
    oil_series,          # pd.Series of oil closes
    strategy,            # Strategy instance (mutated in place)
    initial_capital,     # float, default 100_000
    history_days,        # int, default 60
) -> list[HistoricalBar]
```

Replays the last `history_days` trading bars through `strategy`. Returns one `HistoricalBar` per day with date, price, oil price, drawdown, oil spike, 3-day return, allocation, equity value, and signal label.

```python
simulate_future(
    eq_series,           # pd.Series — used for vol calibration
    oil_series,          # pd.Series — used for vol calibration
    strategy,            # fresh Strategy instance (reset inside)
    last_bar,            # HistoricalBar — starting state for the simulation
    forecast_days,       # int, default 30
    n_paths,             # int, default 500
    seed,                # int, default 42 — for reproducibility
    initial_capital,     # float, default 100_000
) -> ForecastPath
```

Monte Carlo projection. Calibrates daily return distributions from the most recent 20 real bars, then runs `n_paths` independent paths forward. Returns a `ForecastPath` with `dates`, `p10`, `p25`, `p50`, `p75`, `p90` (equity percentiles), and `alloc_median` (median allocation at each future bar).

## Adding a New Strategy

Three steps, all in `strategy.py`:

### 1. Subclass `Strategy`

```python
class MyStrategy(Strategy):
    name  = "My Strategy"   # appears in dashboard tabs and backtest tables
    color = "#a87afc"       # any hex colour

    def next_allocation(self, signals: Signals) -> float:
        # Return a float in [0.0, 1.0]
        # 0.0 = hold cash entirely
        # 1.0 = fully invested
        if signals.drawdown > 0.08 and signals.oil_spike > 0.12:
            return 1.0
        return 0.0
```

Your `next_allocation` method receives only the `Signals` dataclass. It must return a number between 0 and 1. That's the entire contract.

### 2. Register it

```python
REGISTRY: list = [
    OilWarStrategy(),
    BuyOnlyOilWarStrategy(),
    BuyAndHoldStrategy(),
    MyStrategy(),          # ← add here
]
```

### 3. Run

```bash
python3 dashboard.py   # new strategy appears as a sidebar tab
python3 backtest.py    # new strategy appears in every event panel and the overview table
```

No changes to `dashboard.py`, `backtest.py`, or any HTML are needed. Both scripts iterate `REGISTRY` dynamically.

### Stateful strategies

If your strategy needs to track state across bars (like `BuyOnlyOilWarStrategy` tracks shares held), override `reset()` to clear that state, and do your accounting in `next_allocation` or `on_bar`:

```python
class MeanReversionStrategy(Strategy):
    name  = "Mean Reversion"
    color = "#4fa3f7"

    def __init__(self):
        self._entry_price = None

    def reset(self):
        self._entry_price = None   # called before each backtest window

    def next_allocation(self, signals: Signals) -> float:
        # Buy on a significant dip from entry
        if self._entry_price is None:
            self._entry_price = signals.price
        reversion = (signals.price / self._entry_price) - 1
        if reversion < -0.05:
            return 0.75
        if reversion > 0.03:
            return 0.0
        return 0.25

    def on_bar(self, signals: Signals) -> None:
        # Update entry price on strong upward moves
        if signals.return_3d > 0.02:
            self._entry_price = signals.price
```

> **Note:** `BuyOnlyOilWarStrategy` has special accounting that `run_history` and `simulate_future` handle explicitly via `isinstance` checks. For most new strategies, a simple stateless `next_allocation` is sufficient and works with no special engine support.
